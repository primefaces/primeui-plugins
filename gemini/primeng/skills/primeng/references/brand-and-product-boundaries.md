# Brand And Product Boundaries

Use this reference for product naming, licensing, package access or availability, pricing, commercial wording, and customer-facing claims.

## Naming

- Use PrimeUI for the product context and PrimeNG for the Angular component library and its library-specific plugin.
- Describe this skill as the PrimeNG assistant/workflow layer over official MCP sources, not as a documentation bundle or component package.
- Refer PrimeVue and PrimeReact requests to their matching plugins. Do not answer them with Angular code or translate their APIs by analogy.

## Current Sources Only

Use current sources that are actually available:

- PrimeNG MCP output.
- Current official PrimeNG generated docs or guides.
- Current `package.json` metadata.
- Current `LICENSE`, `LICENSE.md`, or official license documentation.
- User-provided official release or policy text.

Do not infer licensing, open-source status, pricing, package access, package availability, release timing, support terms, or commercial-use rights from memory, repository history, package names, or old assumptions.

## Product And Package Questions

1. Identify the exact PrimeNG, theme, icons, MCP, license, or other package being discussed.
2. Inspect current official docs, package metadata, and license sources appropriate to the claim.
3. State only what those sources confirm and distinguish local package presence from public availability.
4. If the current source does not confirm the claim, say so and request official product copy when a definitive customer-facing answer is required.

## Customer-Facing Copy

- Keep claims narrow, current, and attributable to official sources.
- Do not expose license keys or credentials in examples, logs, or responses.
- Do not promise unpublished behavior, compatibility, entitlement, pricing, support, or release timing.
