# Brand And Product Boundaries

Use this reference for product naming, licensing, package access or availability, pricing, commercial wording, and customer-facing claims.

## Naming And Scope

- Use PrimeUI for the product context and PrimeReact for the React component library and its library-specific plugin.
- Use the PrimeReact mode names precisely: `styled`, `tailwind`, `primitive`, and `headless`.
- Describe this skill as the PrimeReact assistant/workflow layer over official MCP sources, not as a documentation bundle, component package, or npm skill package.
- Refer PrimeVue and PrimeNG requests to their matching plugins. Do not answer them with React code or translate their APIs by analogy.

## Current Sources Only

Use current sources that are actually available:

- PrimeReact MCP output for the selected mode.
- Current official PrimeReact generated docs or guides for the selected mode.
- Current `package.json` metadata for the exact package being discussed.
- Current `LICENSE`, `LICENSE.md`, or official license documentation.
- User-provided official release or policy text.

Do not infer licensing, open-source status, pricing, package access, package availability, release timing, support terms, or commercial-use rights from memory, repository history, package names, or old assumptions.

## Product And Package Questions

1. Identify the exact PrimeReact runtime, theme, icon, registry, MCP, or other package or asset being discussed.
2. Inspect current official docs, package metadata, and license sources appropriate to the claim.
3. Distinguish a local package or copied component from a publicly installable package.
4. State only what current sources confirm. If the source is missing, say so and request official product copy when a definitive customer-facing answer is required.

## Customer-Facing Copy

- Keep claims narrow, current, and attributable to official sources.
- Do not expose license keys, credentials, or private registry data in examples, logs, or responses.
- Do not promise unpublished behavior, compatibility, entitlement, pricing, support, or release timing.
- Do not describe a mode-specific result as applying to every PrimeReact mode.
